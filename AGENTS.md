# AI 운영 가이드

이 문서는 `google-chat-archive-viewer`에서 작업하는 AI 에이전트의 기본 행동 규칙을 정의한다.

## 1. 작업 원칙

- 먼저 이슈 요구사항, 현재 변경사항, 관련 문서를 확인한 뒤 작업을 시작한다.
- 요구사항이 불명확하면 추정 내용을 작업 로그나 이슈 코멘트에 남긴다.
- 기존 사용자 변경사항은 되돌리지 않는다.
- 큰 변경 전에 영향 범위와 검증 방법을 짧게 정리한다.

## 2. 문서 확인 순서

작업 시작 시 아래 순서로 확인한다.

1. 현재 이슈 설명과 최근 코멘트
2. 이 파일 `AGENTS.md`
3. `agent-docs/README.md`
4. 작업과 직접 관련된 세부 문서

## 3. 코드 작업 규칙

- 작은 단위로 수정하고, 변경 이유가 드러나는 파일명과 구조를 유지한다.
- 새 설정, 스크립트, 운영 규칙을 추가하면 관련 문서도 함께 갱신한다.
- 자동 생성 파일을 수정했다면 생성 방법 또는 재생성 방법을 문서화한다.

## 4. 검증 규칙

- 범위에 맞는 최소 검증을 우선한다.
- 실행한 검증 명령과 결과를 이슈 코멘트에 남긴다.
- 테스트를 못 돌렸다면 이유와 남은 리스크를 명시한다.

## 5. 커뮤니케이션 규칙

- 작업 종료 전 반드시 이슈 코멘트에 현재 상태, 산출물, 다음 액션을 남긴다.
- 추가 확인이 필요한 경우 질문만 남기지 말고, 가정 기반 초안이나 대안을 함께 제시한다.
- 병렬 작업이 필요하면 하위 이슈로 분리한다.

## 6. 관련 문서

- 운영 개요: `agent-docs/README.md`
- 하네스 작업 방식: `agent-docs/harness.md`
- 문서 운영 규칙: `agent-docs/documentation.md`
