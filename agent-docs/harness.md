# Harness 작업 가이드

이 문서는 하네스를 통해 `google-chat-archive-viewer` 작업을 수행할 때의 기본 절차를 정리한다.

## 1. 시작 전 확인

- 현재 이슈의 목표와 완료 조건을 확인한다.
- 워크트리에 이미 변경된 파일이 있는지 확인한다.
- 새 문서 또는 새 파일 생성 위치가 기존 규칙과 맞는지 확인한다.

## 2. 기본 작업 흐름

1. 이슈 요구사항 요약
2. 관련 파일과 문서 탐색
3. 최소 범위 구현 또는 문서 작성
4. 가능한 범위의 검증 수행
5. 이슈 코멘트로 결과와 다음 액션 공유

## 3. 하네스에서 특히 주의할 점

- 단순 상태 보고만 남기지 말고 실제 산출물을 만든다.
- 장시간 대기 대신 하위 이슈 생성, 블로커 명시, 다음 액션 정의 중 하나를 남긴다.
- 다른 사람이 만든 변경은 명시적 요청 없이 되돌리지 않는다.
- 테스트 범위는 변경 크기에 맞춘다. 작은 문서 변경에 전체 빌드를 강제하지 않는다.

## 4. 검증 예시

- 문서 변경: 링크, 경로, 참조 대상 확인
- 스크립트 변경: 대상 명령 또는 사용 예시 실행
- 설정 변경: 영향받는 실행 경로와 필요한 환경 변수 점검

## 5. 후속 확장 후보

프로젝트가 실제 코드베이스로 확장되면 아래 내용을 추가한다.

- 개발 서버 실행 방법
- 테스트 명령 모음
- 배포 전 체크리스트
- 장애 대응 및 롤백 절차
