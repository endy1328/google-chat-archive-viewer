# 문서 운영 규칙

이 문서는 AI 운영 문서를 유지보수하는 방법을 정의한다.

## 문서 작성 기준

- 독자가 바로 행동할 수 있게 목적, 절차, 예외를 분리해서 쓴다.
- 실제 확인되지 않은 내용은 "가정", "예시", "초안"으로 표시한다.
- 명령어, 경로, 환경 변수는 복사 가능한 형태로 유지한다.

## 문서 갱신이 필요한 경우

- 새 작업 규칙이 생겼을 때
- 하네스 사용 방식이 바뀌었을 때
- 반복되는 리뷰 코멘트나 운영 실수가 발생했을 때
- 신규 팀원이 바로 이해하기 어려운 암묵지가 발견됐을 때

## 리뷰 체크리스트

- 문서 위치가 적절한가
- 기존 규칙과 충돌하지 않는가
- 추상 설명만 있고 실행 단계가 빠지지 않았는가
- 후속 갱신이 필요한 초안 항목이 표시되어 있는가

## 권장 운영

- 기능 문서는 기능 디렉토리 근처에 둔다.
- 저장소 공통 규칙은 `AGENTS.md`에 둔다.
- 하네스, 운영, 온보딩 문서는 `agent-docs/` 아래에 둔다.
